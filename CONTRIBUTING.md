Thank you for your interest in contributing. This document outlines how to suggest changes, report issues, or submit improvements.

## How to contribute

1. **Report bugs** – Open an issue describing the bug, steps to reproduce, and expected behavior.
2. **Suggest features** – Open an issue with a clear description and use case.
3. **Improve docs** – Fix typos, clarify wording, or add examples in README, USAGE, or inline comments.
4. **Submit code** – Follow the guidelines below before sending a pull request.

## Code guidelines

- **CSS:** Use existing design tokens (`var(--rsk-*)`) where possible. Keep class prefix `rsk-`. Add the RSK World comment block at the top of new CSS files.
- **HTML:** Use semantic markup and ARIA where relevant. Keep examples in `index.html`, `components.html`, `examples.html` in sync with new components.
- **Files:** Do not remove or alter the author/copyright comments in existing files.

## Project structure

- `css/` – All stylesheets; `design-system.css` is the main entry (import order matters).
- `*.html` – Demo and documentation pages.
- `*.md` – Documentation (README, USAGE, CONTRIBUTING, CHANGELOG, etc.).
